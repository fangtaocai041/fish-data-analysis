# 📋 证据溯源与引用规范

> 科研工作的底线：**每个结论都有来源，每个来源都可追溯**

---

## 一、信息来源分级

| 等级 | 标识 | 说明 | 适用 |
|------|------|------|------|
| **T0** | ✅ 已核实 | 本人读过全文、可确认内容准确 | 论文核心结论 |
| **T1** | 📚 有出处 | 有DOI/ISBN可查的文献或教材 | 方法选择依据 |
| **T2** | 🔗 可追溯 | 官方文档、机构报告、权威数据库 | 数据来源 |
| **T3** | 🧪 待验证 | 预印本、会议论文、个人通信 | 前沿探索 |
| **T4** | ⚠️ 需警惕 | AI生成、未经核实的网络信息 | 仅作参考 |

## 三、禁止事项

| ❌ 禁止 | ✅ 替代 |
|---------|--------|
| "研究表明…" 无引用 | "XXX (2023) 研究表明…" |
| "常用方法…" 无来源 | "按 Legendre (2012) 的建议…" |
| 使用AI生成的分析结果而不验证 | 仅将AI作为辅助，结果需人工核对 |
| 引用自己没有读过的文献 | 只引用已读全文的文献（可用"转引自"） |
| 使用无版本号的软件/R包 | 记录 sessionInfo() 或 renv.lock |

## 四、参考管理建议

1. **Zotero**：课题组已有 Zotero 文献库（`scripts/zotero_manager.py`）
2. **DOI 优先**：引用时优先使用 DOI，而非 URL
3. **代码注释**：分析脚本中关键步骤需注释方法来源
4. **sessionInfo()**：每次分析输出附 R 环境信息
