## 一、零模型：保留什么、破坏什么？

### 黄金法则

```
保留的特征越多 → 零模型越保守 → 假阴性率越高
保留的特征越少 → 零模型越宽松 → 假阳性率越高

你需要在两者之间找到平衡。

最常犯的错：选了一个过于保守的零模型
→ 真实效应存在但零模型未检出 → "没有显著格局"（Type II error）
→ 但你的数据明明显示有格局！
```

### 群落共现：哪个零模型？

```
quasiswap / curveball：保留行和+列和 → 中等保守 → 推荐首选
r00：保留行列和 → 同 quasiswap，但算法不同
SIM2：只保留出现频率 → 宽松 → 易出假阳性
FF：最宽松 → 不推荐

争论点：
→ 有人批评 SIM2 (Gotelli 2000)
→ 有人批评 quasiswap 过于保守 (Ulrich & Gotelli 2012)
→ 最佳实践：同时报告两种零模型的结果
```

## 二、贝叶斯：先验的选择不是任意的

### 必须报告的内容

```
贝叶斯论文中被审稿人要求但常被遗漏的：

1. ✅ 先验是什么？为什么选这个？
2. ✅ 后验对先验的敏感性 → 换一个先验，结论变吗？
3. ✅ R-hat < 1.01（链收敛诊断）
4. ✅ 有效样本量 > 1000
5. ✅ LOO-CV 或 WAIC（模型比较）
6. ✅ 后验预测检查（posterior predictive check）

如果上述任何一项没有被报告 → 审稿人几乎一定会要求补
```

### brms vs rstanarm

```
推荐 brms：
→ 公式接口（类似 lme4），学习成本低
→ 自动生成 Stan 代码
→ 丰富的分布族（零膨胀、hurdle、beta...）
→ 优秀的文档和社区

rstanarm：更简单，但分布族有限
rjags：老派，需要手写 BUGS 代码（不推荐新手）
```

## 三、Bootstrap 的陷阱

```
❌ 对时间序列做普通 bootstrap
→ 时间自相关被破坏 → CI 过窄 → 过度自信

✅ 对时间序列用 block bootstrap
→ 保持时间块内部的依赖结构

❌ n=10 时做 bootstrap
→ bootstrap 分布不连续 → CI 不可靠

✅ n < 15 时考虑贝叶斯（用先验正则化）
```
