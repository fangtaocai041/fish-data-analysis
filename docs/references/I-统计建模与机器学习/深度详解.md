## 一、GLM：响应变量类型决定一切

### 计数数据的"过离散"：最常见也最严重的问题

```
Poisson GLM 假设：方差 = 均值
现实：方差几乎总是 > 均值 → 过离散

过离散的后果：
→ 标准误被严重低估（可能差 2-5 倍！）
→ p 值虚低 → 假阳性泛滥
→ 论文发表后被撤稿的常见原因之一

检测：dispersion = residual deviance / residual df
  > 1.5 → 换负二项或 quasi-Poisson
  > 2.0 → 必须换，Poisson 完全不可靠
```

### 零膨胀：零值过多怎么办？

```
零值有两种来源：
1. "结构性零"：该样方根本不可能是正数（如物种根本不在该区域）
2. "抽样零"：物种在，但没采到

零膨胀模型：
→ Hurdle 模型：先判断"是否出现"，再对正数建模
→ Zero-inflated 模型：混合分布（零 + 计数分布）

什么时候用？
→ 零的比例 > 30% 且超出分布预期 → 零膨胀模型
→ R: glmmTMB(..., family = nbinom2, ziformula = ~1)
```

## 二、GLMM：随机效应需要多少组？

```
组数 < 5  → 随机效应方差的估计极不稳定
            → 可能估计为 0 或极端值
            → 改用固定效应（虽不优雅但更可靠）

组数 5-10 → 随机效应 OK，但需谨慎解释方差估计
组数 > 10 → 随机效应可靠

检查：
→ summary(model) 中的随机效应方差 ≈ 0？
→ 如果方差接近 0 → 可能模型过于复杂
→ 简化随机效应结构：先只用随机截距，再加斜率
```

## 三、GAM：平滑参数的选择

```
mgcv 默认用 GCV 选平滑度，但 REML 通常更好：
→ gam(y ~ s(x), method = "REML")

检查：
→ gam.check(model) → k 是否太小？
→ k-index < 1 且 p < 0.05 → 增加 k
→ 不要过度担心"p 值太小"——GAM 用于探索趋势，而非假设检验
```

## 四、ML vs 统计建模：互补不是竞争

| 场景 | 推荐 |
|:-----|:-----|
| "什么因素驱动了变化？" | GLM / GLMM / GAM |
| "我们需要最优预测" | 随机森林 / GBM |
| "样本量小但需因果推断" | GLM（每参数10-20观测即可） |
| "样本量大但关系复杂" | ML 或 GAM |
| "需要可解释的论文结果" | GLM / GAM + ML 辅助 |

## 📋 I类诊断清单

```
□ 计数数据 → 是否检查了过离散？
□ 零值 > 30% → 是否考虑了零膨胀模型？
□ GLMM → 随机效应组数 ≥ 5 吗？
□ GAM → k 是否通过 gam.check()？
□ ML → 是否在 Discussion 中讨论了可解释性 vs 精度？
```
