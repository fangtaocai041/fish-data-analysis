## C1 — 排序分析：完整决策框架

### 一、排序方法论战图

```
                    你有物种数据？
                         │
        ┌────────────────┼────────────────┐
        ↓                ↓                ↓
   只有物种数据     物种+环境变量     只有距离矩阵
   (探索性)         (解释性)          (如遗传距离)
        │                │                │
   DCA 检查梯度          │           PCoA / NMDS
        │                │
   < 3 SD?  > 4 SD?     │
     ↓        ↓         │
    PCA    CA/DCA       │
   RDA      CCA  ←──────┘
```

### 二、梯度长度检测：被忽视的"守门人"

```
为何必须先跑 DCA？
→ DCA 第一轴长度告诉你物种响应是线性(<3SD)还是单峰(>4SD)
→ 选错 = 整个分析建立在不合适的假设上

真实案例：
研究者跑了 RDA，审稿人问"为什么不跑 CCA？"
→ 因为你没报告 DCA 第一轴长度！
→ 先跑 DCA → 报告 SD < 3 → 有理有据选择 RDA → 审稿人满意
```

### 三、五种排序方法深度对比

| | PCA | CA/DCA | NMDS | RDA | CCA |
|:--|:---|:------|:-----|:----|:----|
| **输入数据** | 环境变量 | 群落矩阵 | 任意距离 | 群落+环境 | 群落+环境 |
| **响应假设** | 线性 | 单峰 | 无 | 线性 | 单峰 |
| **距离** | 欧氏 | χ² | 任意 | 欧氏(→Hellinger!) | χ² |
| **解释方差** | ✅ | ✅ | ❌ | ✅ | ✅ |
| **VPA可用** | ✅ | ❌ | ❌ | ✅ | ✅ |
| **p值** | ❌ | ❌ | ❌ | ✅ 置换 | ✅ 置换 |
| **可视化友好** | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **双零免疫** | ❌→需Hellinger | ✅ | 取决于距离 | ❌→需Hellinger | ✅ |

### 四、NMDS 的 Stress 值：不止于 <0.2

```
Stress 值解读：

< 0.05 → 优秀（排序几乎完美代表原始距离）
0.05-0.10 → 良好（可用于发表）
0.10-0.20 → 可接受（可能需要解释）
0.20-0.30 → 勉强（必须在论文中讨论局限性）
> 0.30 → 不可用（考虑增加维度或换方法）

但要知道：
- Stress 随维度增加而降低（k=3 总是比 k=2 低）
- Stress 随样方数减少而降低（n=5 时 Stress 总是很低，但没有意义）
- 不要为了低 Stress 增加太多维度（难以可视化）
```

### 五、RDA 变量选择的陷阱

```r
# ❌ 跑前向选择然后只报告"显著"的变量
# ✅ 应该：
# 1. 先基于生态学知识选入核心变量
# 2. 用前向选择作为辅助
# 3. 报告 adjusted R² 而非 R²
# 4. 讨论未入选但生态学上重要的变量

# 关键：ordiR2step（vegan≥2.6推荐，替代旧ordistep）的p值未校正多重比较！
# 一个"显著"的变量可能只是 20 次测试中的幸运儿
```

### 六、方差分解 (VPA) 的负 R² 问题

```
VPA 可能出现"负的方差解释" — 不要慌

负 adjusted R² 的含义：
→ 该变量组解释的方差 < 随机噪声能解释的
→ 本质上 = 0（不提供独特信息）

处理方式：
→ 在论文中报告负值，并在图中标注为 0
→ 或直接写"< 可忽略"

不要做的事：
→ 不说负数等于 0（这是数据造假）
→ 过度解释负数
```

### C1 诊断代码

```r
# ============================================================
# C1 增强版：排序方法选择 + 完整诊断
# ============================================================
library(vegan)
library(ggfortify)   # ggplot2 风格的 vegan 自动画图 (autoplot)
library(ggrepel)    # 标签防重叠

# ---- 1. 梯度长度决策函数 ----
decide_ordination <- function(spe_raw) {
  # ⚠️ DCA 必须在原始群落数据（非Hellinger转换后）上运行！
  dca <- decorana(spe_raw)
  axis1_sd <- dca$evals[1]
  
  cat("\n=== 排序方法决策 ===\n")
  cat(sprintf("DCA 第一轴长度: %.2f SD\n", axis1_sd))
  
  if (axis1_sd < 3) {
    cat("→ ✅ 推荐 RDA / PCA（线性方法）\n")
    cat("短梯度意味着物种对环境的响应是近似线性的\n")
    return("linear")
  } else if (axis1_sd > 4) {
    cat("→ ✅ 推荐 CCA / CA（单峰方法）\n")
    cat("长梯度意味着物种沿环境梯度有最优值\n")
    return("unimodal")
  } else {
    cat("→ ⚠️ 梯度长度在过渡区 (3-4 SD)\n")
    cat("→ 建议两种方法都跑，选择解释率更高的\n")
    cat("→ 或者在论文中同时报告，讨论差异\n")
    return("transition")
  }
}

# ---- 2. NMDS 稳健性检查 ----
check_nmds_stress <- function(dist_mat, max_k = 4, n_tries = 50) {
  results <- data.frame(k = 1:max_k, stress = NA, converged = NA)
  
  for (k in 1:max_k) {
    stresses <- numeric(n_tries)
    for (i in 1:n_tries) {
      nmds_i <- try(metaMDS(dist_mat, k = k, try = 1, trace = 0), silent = TRUE)
      if (!inherits(nmds_i, "try-error")) {
        stresses[i] <- nmds_i$stress
      }
    }
    results$stress[k] <- min(stresses[stresses > 0], na.rm = TRUE)
    results$converged[k] <- sum(stresses > 0, na.rm = TRUE) / n_tries
  }
  
  cat("\n=== NMDS Stress 诊断 ===\n")
  print(results)
  
  # 肘部法则选 k
  # k=2 通常是首选（可可视化），除非 stress 太高
  return(results)
}

# ---- 3. RDA/CCA 变量共线性预警 ----
check_collinearity <- function(rda_obj, threshold = 10) {
  vif_vals <- vif.cca(rda_obj)
  cat("\n=== 共线性诊断 (VIF) ===\n")
  
  high_vif <- vif_vals[vif_vals > threshold]
  if (length(high_vif) > 0) {
    cat("⚠️ VIF >", threshold, "的变量:\n")
    print(high_vif)
    cat("建议：逐步删除 VIF 最大的变量，直到所有 VIF <", threshold, "\n")
  } else {
    cat("✅ 所有变量 VIF <", threshold, "\n")
  }
  return(vif_vals)
}

# ---- 4. 置换检验的可靠性 ----
# 置换次数影响 p 值的精度
# 999 次置换 → p 的最小非零值是 0.001
# 如果需要 p < 0.001，用 9999 次
# 置换检验的 p 值 = (n_extreme + 1) / (n_perm + 1)
```

## C2 — 组间差异检验：PERMANOVA 完整链路

### 一、为什么 PERMANOVA 是黄金标准（而非 ANOSIM 或 MRPP）？

| | PERMANOVA | ANOSIM | MRPP |
|:--|:---------|:------|:-----|
| **理论基础** | 基于距离的方差分解 | 基于秩 | 基于距离的置换 |
| **处理复杂设计** | ✅ 多因素+交互 | ❌ 仅单因素 | ⚠️ 有限 |
| **统计效力** | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ |
| **当代推荐** | **首选** | 不推荐（效力低） | 备选 |

### 二、PERMANOVA 完整诊断流水线

```
Step 1: betadisper（同质性检验）
  ├─ p > 0.05 → ✅ 通过 → 继续
  └─ p < 0.05 → ⚠️ 警告！组内离散度不同

Step 2: PERMANOVA
  ├─ p < 0.05 → 组间有显著差异
  └─ 但！如果 betadisper 也显著 → 
      差异可能来自离散度，不是位置

Step 3: 事后比较
  ├─ 多组 → pairwise adonis（需 Bonferroni 校正）
  └─ 报告校正后的 p 值

Step 4: SIMPER / IndVal
  └─ 找出哪些物种驱动了差异
```

### 三、PERMANOVA 的局限性

```
1. 不能区分位置和离散度的差异
   → 必须同时报告 betadisper

2. 对不平衡设计敏感
   → 各组 n 相差很大时，用 adonis2(..., by = "margin")

3. 不能处理复杂的协方差结构
   → 重复测量 → 需要专门的 PERMANOVA 扩展
   → 或用 GLMM 替代

4. 只能检验组间差异，不能建模连续梯度
   → 连续环境变量 → 用 RDA/CCA 而非 PERMANOVA
```

### C2 诊断代码

```r
# ============================================================
# C2 增强版：PERMANOVA 完整诊断链路
# ============================================================
library(vegan)
library(pairwiseAdonis)  # 事后比较

# ---- 完整 PERMANOVA 流程 ----
permanova_pipeline <- function(spe, group, dist_method = "bray", 
                                nperm = 9999, correct_p = TRUE) {
  
  # 距离矩阵
  dist_mat <- vegdist(spe, method = dist_method)
  
  # === Step 1: betadisper ===
  cat("\n========== Step 1: 同质性检验 ==========\n")
  bd <- betadisper(dist_mat, group)
  bd_anova <- anova(bd)
  cat("betadisper p =", round(bd_anova$`Pr(>F)`[1], 4), "\n")
  
  if (bd_anova$`Pr(>F)`[1] < 0.05) {
    cat("⚠️ 组内离散度不相等！\n")
    cat("PERMANOVA 显著需谨慎解释——可能是离散度差异\n")
    cat("\n各组到质心的平均距离:\n")
    print(tapply(bd$distances, bd$group, mean))
  } else {
    cat("✅ 组内离散度齐性\n")
  }
  
  # === Step 2: PERMANOVA ===
  cat("\n========== Step 2: PERMANOVA ==========\n")
  perm_result <- adonis2(dist_mat ~ group, permutations = nperm)
  print(perm_result)
  
  # === Step 3: 事后比较 ===
  cat("\n========== Step 3: 配对比较 ==========\n")
  if (length(unique(group)) > 2) {
    pw <- pairwise.adonis(dist_mat, group, p.adjust.method = "bonferroni")
    print(pw)
  } else {
    cat("只有两组，不需要事后比较\n")
  }
  
  # === Step 4: SIMPER（关键差异物种） ===
  cat("\n========== Step 4: SIMPER ==========\n")
  simp <- simper(spe, group, permutations = 999)
  # 只看最重要的前5个物种
  for (comp_name in names(simp)) {
    cat("\n---", comp_name, "---\n")
    top5 <- head(summary(simp[[comp_name]], ordered = TRUE), 5)
    top5$species <- rownames(top5)  # SIMPER返回的物种名在行名中
    print(top5[, c("species", "average", "sd", "ratio", "ava", "avb", "p")])
  }
  
  # === 返回诊断报告 ===
  return(list(
    betadisper = bd,
    permanova = perm_result,
    pairwise = if (exists("pw")) pw else NULL,
    simper = simp
  ))
}
```

## C3 — 指示种分析：IndVal 详解

### IndVal 的双重标准

```
IndVal = A × B × 100

A = 特异性 (Specificity)：该物种在目标组中的"专属度"
    A = 1.0 → 该物种只出现在目标组
    A = 0.5 → 该物种在目标组和其他组中平均分布

B = 保真度 (Fidelity)：该物种在目标组中出现的频率
    B = 1.0 → 目标组的所有样方都有该物种
    B = 0.5 → 目标组的一半样方有该物种

因此：
IndVal 高 ↔ 物种在目标组中既"专一"又"常见"

适合做指示种的物种：
  IndVal > 0.7 且 p < 0.05（严格）
  IndVal > 0.5 且 p < 0.05（宽松，用于探索）
```

### IndVal vs SIMPER

| | IndVal | SIMPER |
|:--|:------|:------|
| **目标** | 找出每组"专属"的物种 | 找出组间差异的贡献者 |
| **标准** | 特异性 + 保真度 | 对 Bray-Curtis 差异的贡献 |
| **输出** | 每组的特征物种 | 组对的差异物种 |
| **适合** | 生态指示、保护生物学 | 解释群落差异 |

## C4 — 聚类分析在生态学中的特殊考量

### 为什么生态学聚类不能用默认参数？

```
问题1: 群落距离矩阵 ≠ 欧氏距离
  → hclust(dist(spe)) 在群落数据上是错误的！
  → 先用 vegdist() 再用 hclust()

问题2: 聚类数 k 的确定没有金标准
  → 生态学中 silhouette width 和 gap statistic 可能给出不同答案
  → 最终 k 的选择应基于生态学可解释性

问题3: 层次聚类算法选择
  → UPGMA (average linkage): 生态学最常用，假设等速率变化
  → Ward's method: 倾向于产生等大小的类（但不一定"正确"）
  → Complete linkage: 对异常值敏感
```

### 聚类验证

```r
# ---- 聚类稳健性检查 ----
library(cluster)
library(fpc)     # cluster.stats
library(clValid) # 聚类验证

# 1. 用 silhouette 选 k
sil_width <- numeric(10)
for (k in 2:10) {
  clust <- pam(bc_dist, k = k)
  sil_width[k] <- clust$silinfo$avg.width
}
# → 最高的 silhouette 对应的 k（但也看生态学意义）

# 2. 比较多种聚类算法
# 如果 UPGMA、Ward、PAM 给出相似的分类 → 稳健
# 如果差异大 → 结构可能不清晰
```

## 📋 C类完整诊断清单

```
□ 1. 排序前检查
    □ 群落数据是否做了 Hellinger（用PCA/RDA时）？
    □ DCA 第一轴 SD 是否已报告？
    □ 环境变量是否标准化？

□ 2. NMDS 检查
    □ Stress 值是多少？是否报告了？
    □ 是否多次尝试（try=100+）避免局部极小？

□ 3. RDA/CCA 检查
    □ VIF 是否全部 < 10？
    □ 前向选择是否报告了 adjusted R²？
    □ 是否用了置换检验评估显著性（999次以上）？

□ 4. VPA 检查
    □ 负 adjusted R² 是否如实报告？
    □ 变量组数是否合理（每组变量数 < n/3）？
    □ 各组部分的显著性是否独立检验？

□ 5. PERMANOVA 检查
    □ betadisper 是否在 PERMANOVA 之前做了？
    □ betadisper p > 0.05? 如果 < 0.05 是否讨论了？
    □ 多组比较是否做了事后检验 + 多重比较校正？

□ 6. 聚类检查
    □ 是否使用了正确的距离矩阵？
    □ k 的选择是否有生态学理由（不仅是统计最优）？
    □ 聚类结果的稳健性是否验证了？

□ 7. 常见审稿质疑（提前预防）
    □ "为什么不选 CCA 而选 RDA？" → 报告 DCA 第一轴 SD
    □ "NMDS Stress 0.18，结果可靠吗？" → 附上 Shepard 图
    □ "变量选择方法是否过度拟合？" → 用 adjusted R² 回答
    □ "PERMANOVA 的差异来自离散度还是位置？" → 附 betadisper 结果
```
