## E1 — MaxEnt：最流行的 SDM 方法详解

### 一、MaxEnt 的核心逻辑

```
MaxEnt = 在环境约束下，寻找"最均匀"的分布

简单理解：
输入：存在点的环境条件 + 整个研究区的环境背景
输出：每个像元的"生境适宜度"（0-1，相对值）

关键假设：
→ 存在点代表了物种能生存的环境
→ 背景点代表了"可用环境"的范围
→ 物种分布由提供的气候变量充分描述
```

### 二、MaxEnt 的五个致命陷阱

| 陷阱 | 问题 | 解决方案 |
|:-----|:-----|:---------|
| **采样偏差** | 存在点集中在交通便利处 → 模型学到"靠近公路" | 空间稀疏化 + bias file |
| **默认参数** | 默认 regularization=1, features=auto 不总是最优 | ENMeval 调参 |
| **AUC 虚高** | 背景范围过大 → AUC > 0.95（但模型并不好） | 检查 TSS + Boyce 指数 |
| **环境外推** | 预测到未在训练数据中出现的环境组合 | MOP 分析 |
| **变量相关性** | 高度相关的变量导致模型不稳定 | |r| > 0.8 的变量只保留一个 |

### 三、MaxEnt 调参：不止于默认值

```r
library(ENMeval)
library(dismo)
library(raster)

# ---- 完整 ENMeval 调参流程 ----
# 准备数据
env_stack <- stack(list.files("climate/", pattern = ".tif$", full.names = TRUE))
occs <- read.csv("occurrences.csv")
bg <- read.csv("background.csv")

# ⚡ 关键：用空间分块 CV（block），而非随机 CV！
# 随机 CV 在有空间自相关时会严重低估 omission rate
eval_results <- ENMevaluate(
  occs = occs[, c("longitude", "latitude")],
  envs = env_stack,
  bg.coords = bg[, c("longitude", "latitude")],
  algorithm = "maxnet",
  tune.args = list(
    fc = c("L", "LQ", "H", "LQH", "LQHP"),
    rm = c(0.5, 1, 2, 3, 5)
  ),
  partitions = "block",
  partition.settings = list(orientation = "lat_lon"),
  parallel = TRUE
)

# 选最优模型
best_results <- eval_results@results %>%
  filter(delta.AICc < 2) %>%
  arrange(AICc)
print(best_results[, c("fc", "rm", "AICc", "delta.AICc", "or.10p")])

# ⚠️ 不要只选 ΔAICc=0！ΔAICc<2 的模型都有实质支持
# 最终模型
best <- best_results[1, ]
final_mod <- eval_results@models[[best$tune.args]]

# 预测
pred_current <- predict(final_mod, env_stack, type = "cloglog")
plot(pred_current)

# ---- 变量重要性 ----
plot(final_mod)  # MaxEnt 的响应曲线

# ---- 模型评估（不只靠AUC） ----
# 计算 TSS
library(ecospat)
binom_data <- cbind(rep(1, nrow(occs)), rep(0, nrow(occs)))
pred_occs <- extract(pred_current, occs[, c("longitude", "latitude")])
pred_bg   <- sampleRandom(pred_current, 1000)

# TSS = max(灵敏度 + 特异度 - 1) across thresholds
evaluate_tss <- function(pred_pres, pred_abs, n_thresh = 100) {
  thresh <- seq(0, 1, length.out = n_thresh)
  tss_vals <- sapply(thresh, function(t) {
    sens <- mean(pred_pres >= t)
    spec <- mean(pred_abs < t)
    sens + spec - 1
  })
  return(max(tss_vals))
}
tss <- evaluate_tss(pred_occs, pred_bg)
cat(sprintf("TSS: %.3f\n", tss))
```

### 四、模型评估：AUC 不够！

```
为什么 AUC 会虚高？
→ 把"无人调查的区域"当作"真缺席"
→ 研究区越大，背景点越多，AUC 越高

替代指标组合：
✅ TSS (True Skill Statistic) = 灵敏度 + 特异度 - 1
   → > 0.4 可接受，> 0.6 好，> 0.8 优秀

✅ Boyce 指数（仅存在数据时）
   → [-1, 1]，正值=优于随机，> 0.5 好

✅ Omission Rate
   → 使用最低存在阈值 (LPT)，应接近预设的 omission rate
```

## E2 — 生态位分析（ecospat）

### 一、生态位等价性 vs 相似性：不要混淆

```
等价性检验 (Equivalency test)
问题："两个物种的生态位是否完全相同？"
方法：将两个物种的存在点混合后随机分为两组 → 置换检验
H0：生态位等价（共享同一生态位）
→ p < 0.05 → 拒绝等价 → 生态位不同！

相似性检验 (Similarity test)
问题："物种A的生态位是否比随机预期更相似于物种B？"
方法：将物种A的生态位与背景的随机生态位比较
H0：物种A从物种B的背景中随机抽取
→ p < 0.05 → 两个生态位比随机预期更相似

常见错误：用等价性检验的 p > 0.05 说"生态位相同"
→ 可能只是统计效力不够（n太小）
→ 应该用 effect size + 置信区间
```

### 二、Schoener's D 和 Hellinger's I

| | Schoener's D | Hellinger's I |
|:--|:------------|:-------------|
| **范围** | 0 (完全不同) 到 1 (完全相同) | 0 到 1 |
| **对稀有生境的敏感度** | 中等 | 较高 |
| **推荐** | 更常用 | 作为补充 |

### 三、ecospat 完整操作代码

```r
library(ecospat)
library(ade4)

# ---- 生态位重叠 + 等价性/相似性检验 ----
# Step 1: 环境 PCA
env_all <- na.omit(values(env_stack))
pca_env <- dudi.pca(env_all, scannf = FALSE, nf = 2)

# Step 2: 两个物种在 PCA 空间中的分布
scores_glob <- pca_env$li
spA_scores <- suprow(pca_env, extract(env_stack, occs_A))$li
spB_scores <- suprow(pca_env, extract(env_stack, occs_B))$li

# Step 3: 核密度估计（构建生态位"面"）
grid_env <- ecospat.grid.clim.dyn(
  glob = scores_glob,
  glob1 = scores_glob,
  sp = spA_scores,
  R = 100
)

spA_grid <- ecospat.grid.clim.dyn(
  glob = scores_glob, glob1 = scores_glob,
  sp = spA_scores, R = 100
)$z.uncor

spB_grid <- ecospat.grid.clim.dyn(
  glob = scores_glob, glob1 = scores_glob,
  sp = spB_scores, R = 100
)$z.uncor

# Step 4: 生态位重叠 (Schoener's D)
niche_D <- ecospat.niche.overlap(spA_grid, spB_grid, method = "D")
niche_I <- ecospat.niche.overlap(spA_grid, spB_grid, method = "I")
cat(sprintf("Schoener's D = %.3f, Hellinger's I = %.3f\n", niche_D, niche_I))

# Step 5: 等价性检验（H0: 两个生态位完全相同）
eq_test <- ecospat.niche.equivalency.test(
  z1 = spA_grid, z2 = spB_grid,
  rep = 999, ncores = 4
)
cat(sprintf("等价性检验 p = %.3f\n", eq_test$p.D))

# Step 6: 相似性检验（H0: 物种A从物种B的背景随机抽取）
sim_test <- ecospat.niche.similarity.test(
  z1 = spA_grid, z2 = spB_grid,
  rep = 999, alternative = "greater", ncores = 4
)
cat(sprintf("相似性检验 p = %.3f\n", sim_test$p.D))

# 可视化
plot(eq_test$z)  # 等价性检验结果（观测 vs 零分布）
```

## E3 — SDM 的"可转移性"问题

### 为什么大多数 SDM 不能外推？

```
问题：训练环境 ≠ 预测环境

例：用当前中国气候数据训练的模型 → 预测 2080 年
→ 2080 年可能出现"当前中国完全不存在"的气候组合
→ 模型在这些新环境中的预测 = 外推 = 不可靠

解决方案：
1. MOP (Multivariate Environmental Similarity Surface)
   → 标记出"新环境"区域 → 这些区域的预测不要用
2. 使用多个 GCM（气候模型）取集合
3. 报告预测的不确定性（而非单一预测）
```

## 📋 E类完整诊断清单

```
□ 1. 存在点
    □ 是否做了空间稀疏化（去除空间聚集的冗余点）？
    □ 存在点 ≥ 20? ≥ 50 更好
    □ 是否存在采样偏差？（检查道路、城市附近的聚集）

□ 2. 环境变量
    □ |r| > 0.8 的变量对是否只保留了一个？
    □ 变量选择是否有生态学依据？
    □ 分辨率与研究问题匹配吗？

□ 3. MaxEnt
    □ 是否调了 feature classes + regularization？
    □ 是否用了空间分块 CV（而非随机 CV）？
    □ 是否报告了 AUC + TSS + Boyce 指数？

□ 4. 预测
    □ 是否做了 MOP 分析？
    □ 是否使用了多模型集合？
    □ 是否报告了预测不确定性？

□ 5. 生态位检验
    □ 等价性 vs 相似性是否正确区分？
    □ 环境空间定义是否合理（PCA 轴数、网格分辨率）？
```
