## A1 — 异常值检测（5方法全方位对比）

### 为什么异常值检测如此重要？

一个异常值就能：
- 改变回归斜率 30-50%（Cook's D >> 1 的点）
- 扭曲 PCA 的第一轴方向（Mahalanobis 距离会揭示）
- 让 Box-Cox 的 λ 估计完全偏离最优值

### 方法对比矩阵

| 方法 | 检测维度 | 假设 | 小样本 n<20 | 中样本 20-100 | 大样本 >100 | 偏态数据 | 多变量 |
|:-----|:---------|:-----|:-----------|:-------------|:-----------|:--------|:------|
| **boxplot IQR** | 单变量 | 近似对称 | ⚠️ 阈值不稳 | ✅ | ✅ | ❌ 假阳性多 | ❌ |
| **adjusted boxplot** | 单变量 | 无 | ⚠️ | ✅ | ✅ | ✅ medcouple校正 | ❌ |
| **Cook's D** | 回归影响 | 线性模型 | ⚠️ D>4/n 阈值不稳 | ✅ | ✅ | ❌ 需先建模 | ✅ 间接 |
| **Mahalanobis D²** | 多变量 | 多元正态 | ❌ n<3p不可逆 | ⚠️ | ✅ | ⚠️ | ✅ |
| **Robust Mahalanobis** | 多变量 | 无 | ❌ | ⚠️ | ✅ | ✅ MCD估计 | ✅ |
| **LOF** | 多变量局部 | 无 | ⚠️ | ✅ | ✅ | ✅ | ✅ |

### 决策树：什么时候用什么？

```
数据类型？
├─ 单变量（逐个检查）
│   ├─ 近似正态 → boxplot IQR 规则
│   ├─ 偏态分布 → adjusted boxplot (medcouple)
│   └─ 回归模型上下文中 → Cook's D + DFFITS
│
├─ 多变量（联合检查）
│   ├─ n > 3p 且近似正态 → Mahalanobis D²
│   ├─ n > 3p 且偏态 → Robust Mahalanobis (MCD)
│   ├─ n < 3p → LOF (局部离群因子)
│   └─ 高维 p > n → PCA前几个主成分的 Mahalanobis
│
└─ 极高维（如代谢组学）
    └─ Isolation Forest / One-Class SVM
```

### 异常值处理策略（不是自动删除！）

```
发现异常值后：
│
├─ 录入错误？（pH=14.5, temp=-50°C）
│   └─ → 核对原始记录 → 修正或删除
│
├─ 极端但真实的值？（稀有物种大爆发）
│   └─ → 保留！但要：
│       ├─ 做敏感性分析（有 vs 无该点，结果差多少？）
│       ├─ 在论文中报告并解释
│       └─ 考虑用稳健方法（如稳健回归）替代
│
└─ 方法学伪迹？（网具破损导致"零捕获"）
    └─ → 标记为缺失值，后续用 MICE 插补
```

### R 诊断代码（可直接运行）

```r
# ============================================================
# A1 增强版：多方法异常值检测 + 决策辅助
# ============================================================
library(robustbase)   # adjusted boxplot + MCD
library(dbscan)       # LOF algorithm
library(car)          # Cook's D diagnostic plots

# ---- 1. Adjusted Boxplot（偏态数据专用） ----
# 普通 boxplot 对偏态数据：假阳性可达 5-8%
# Adjusted boxplot 用 medcouple 自动校正偏度
set.seed(42)
x_skewed <- rlnorm(100, meanlog=0, sdlog=1.5)  # 严重偏态
x_skewed[98] <- 50  # 加一个真正的异常值

# 比较两种方法
out_standard <- which(x_skewed > quantile(x_skewed, 0.75) + 1.5*IQR(x_skewed) |
                       x_skewed < quantile(x_skewed, 0.25) - 1.5*IQR(x_skewed))
# adjboxStats 返回 $fences 而非 $out，用 fences 判定
adj_stats <- adjboxStats(x_skewed)
out_adjusted <- which(x_skewed < adj_stats$fences[1] | x_skewed > adj_stats$fences[2])  # 需要转换

cat("标准 boxplot 标记了", length(out_standard), "个异常值\n")
cat("Adjusted boxplot 标记了", length(out_adjusted), "个异常值\n")
# → 偏态时标准 boxplot 通常标记过多！

# ---- 2. Robust Mahalanobis (MCD) ----
# 适合偏态多变量数据
data_multi <- data.frame(
  x1 = rlnorm(100, 0, 1),
  x2 = rnorm(100, 5, 2),
  x3 = rgamma(100, 2, 0.5)
)
data_multi[98:100, ] <- data_multi[98:100, ] * 3  # 加入异常值

# 经典 Mahalanobis（均值+协方差受异常值影响 → 遮蔽效应）
mcd <- covMcd(data_multi)  # MCD 估计
d_classic <- mahalanobis(data_multi, colMeans(data_multi), cov(data_multi))
d_robust <- mahalanobis(data_multi, mcd$center, mcd$cov)

# 阈值：χ²(3, 0.975) = 3.06
threshold <- qchisq(0.975, df = 3)
cat("经典方法标记:", sum(d_classic > threshold), "个\n")
cat("MCD 稳健法标记:", sum(d_robust > threshold), "个\n")
# → 经典法可能因"遮蔽效应"漏掉真异常值

# ---- 3. Cook's Distance 诊断图 ----
# 在回归场景中使用
model <- lm(mpg ~ wt + hp, data = mtcars)
# 增强版诊断：同时看 Cook's D + Leverage + 学生化残差
influencePlot(model, id = list(n=3))  # 标记最危险的3个点

# ---- 4. 综合异常值报告函数 ----
comprehensive_outlier_report <- function(df, vars = NULL) {
  if (is.null(vars)) vars <- names(df)[sapply(df, is.numeric)]
  
  report <- data.frame(
    variable = character(),
    n_outliers_iqr = integer(),
    n_outliers_adj = integer(),
    skewness = numeric(),
    recommendation = character()
  )
  
  for (v in vars) {
    x <- df[[v]]
    x_clean <- x[is.finite(x)]
    sk <- abs(moments::skewness(x_clean, na.rm = TRUE))
    
    out_iqr <- sum(x_clean > quantile(x_clean, 0.75, na.rm=TRUE) + 
                   1.5*IQR(x_clean, na.rm=TRUE) |
                   x_clean < quantile(x_clean, 0.25, na.rm=TRUE) - 
                   1.5*IQR(x_clean, na.rm=TRUE))
    
    rec <- ifelse(sk > 1, "用 adjusted boxplot 或 winsorize",
                  ifelse(sk > 0.5, "boxplot + visual check", "boxplot OK"))
    
    report <- rbind(report, data.frame(
      variable = v, n_outliers_iqr = out_iqr, skewness = round(sk, 2),
      recommendation = rec
    ))
  }
  return(report)
}
```

## A2 — 正态性检验与转换（完整决策链）

### 正态性检验不是非黑即白

**核心误区**：p > 0.05 不等于"数据是正态的"。Shapiro-Wilk 在大样本时过于灵敏（几乎总是 p < 0.05），在小样本时不够灵敏（可能漏掉严重偏态）。

### 检验方法对比

| 检验 | 效力 | 样本范围 | 灵敏度 | 弱点 |
|:-----|:-----|:--------|:------|:-----|
| **Shapiro-Wilk** | 最高 | n < 5000 | 全分布 | n > 5000 必显著 |
| **Anderson-Darling** | 高 | 任意 | 尾部 | 对尖峰不敏感 |
| **Kolmogorov-Smirnov** | 低 | 任意 | 中心 | 效力低，不推荐 |
| **Q-Q 图** | 视觉 | 任意 | 全分布 | 需要经验判断 |
| **偏度 + 峰度** | 低 | n > 50 | 形状 | 只能看大致形状 |

### 实战决策树

```
你需要正态性吗？
│
├─ 后续用参数检验（t-test, ANOVA, Pearson 相关）
│   ├─ n < 30 → Shapiro-Wilk
│   │   ├─ p > 0.05 → 可用参数检验
│   │   └─ p < 0.05 → 看 Q-Q 图
│   │       ├─ 严重偏离 → 转换或非参数
│   │       └─ 轻微偏离 → 参数检验也可（中心极限定理不够用，n太小）
│   │
│   └─ n > 30 → 看 Q-Q 图 + 偏度
│       ├─ |skewness| < 1 → 参数检验够稳健
│       └─ |skewness| > 2 → 必须转换或非参数
│
├─ 后续用 GLM → 不需要正态！选对分布族即可
├─ 后续用随机森林 → 完全不需要
└─ 后续用 PERMANOVA → 不需要（置换检验）
```

### 转换方法的选择逻辑

```
数据特征 → 推荐转换
│
├─ 严重正偏（如 NO3 浓度，右尾长）
│   └─ log(x) 或 log(x+c) → 优先
│       └─ 如果还有零值 → log(x+1) 或 log(x + min(x>0)/2)
│
├─ 中等偏态
│   └─ sqrt(x) → 比 log 更温和
│
├─ 计数数据（方差=均值？）
│   ├─ 方差 ≈ 均值 → sqrt(x)
│   └─ 方差 > 均值（过离散） → log(x+1) 或直接 NB GLM
│
├─ 比例/百分比
│   ├─ 0-100% → arcsin(sqrt(x/100))
│   ├─ 0-1 → logit(x) 或 Beta 回归
│   └─ 有很多 0 或 1 → 不要用 arcsin，用 Beta 回归
│
├─ 不确定用什么 λ？
│   └─ Box-Cox（自动选 λ） → 但先去掉异常值！
│       └─ 有零值 → Yeo-Johnson（允许零和负值）
│
└─ 群落数据（特殊！）
    ├─ 用于 PCA/RDA → Hellinger 或 Chord 转换
    ├─ 用于 NMDS/PERMANOVA → 不需要转换，直接用 Bray-Curtis
    └─ 用于聚类 → Hellinger 后用欧氏距离
```

### Box-Cox 的陷阱

```
Box-Cox 的 λ 估计不稳定当：
1. 有异常值 → λ 被拉向 0 或 1
2. n < 20 → λ 的置信区间很宽
3. 数据有多个"团"（混合分布） → λ 没有意义

正确做法：先 winsorize 异常值 → 再 Box-Cox → 检查转换后的 Q-Q 图
```

### R 诊断代码

```r
# ============================================================
# A2 增强版：正态性诊断 + 转换选择
# ============================================================
library(moments)   # skewness, kurtosis
library(car)       # powerTransform, bcPower
library(bestNormalize)  # 自动找最佳正态化转换（强烈推荐）

# ---- 1. 综合正态性诊断 ----
normality_diagnostic <- function(x, name = "variable") {
  x <- x[is.finite(x)]
  n <- length(x)
  
  # 偏度和峰度
  sk <- skewness(x)
  ku <- kurtosis(x)  # excess kurtosis
  
  # Shapiro-Wilk（如果 n < 5000）
  sw_p <- if (n < 5000 && n >= 3) shapiro.test(x)$p.value else NA
  
  # 解释
  skew_flag <- abs(sk) > 1
  kurt_flag <- abs(ku) > 2
  
  cat("\n=== ", name, " 正态性诊断 (n=", n, ") ===\n", sep="")
  cat("偏度:", round(sk, 2), if(skew_flag) "⚠️ 严重偏态" else "✓", "\n")
  cat("峰度:", round(ku, 2), if(kurt_flag) "⚠️ 峰度异常" else "✓", "\n")
  if (!is.na(sw_p)) {
    cat("Shapiro-Wilk p:", round(sw_p, 4),
        if(sw_p < 0.05) "→ 拒绝正态性假设" else "→ 未拒绝正态性", "\n")
  }
  
  # 推荐行动
  if (abs(sk) < 0.5 && abs(ku) < 1) {
    cat("→ ✅ 数据近似正态，可直接用参数方法\n")
  } else if (abs(sk) < 1 && abs(ku) < 2) {
    cat("→ ⚠️ 轻微偏离，参数方法大概率 OK（特别是 n>30）\n")
  } else if (abs(sk) < 2) {
    cat("→ 🔄 建议转换或使用非参数方法\n")
  } else {
    cat("→ 🚫 必须转换或使用非参数/稳健方法\n")
  }
  
  # Q-Q 图
  qqnorm(x, main = paste("Q-Q Plot:", name))
  qqline(x, col = "red", lwd = 2)
  
  invisible(list(skewness = sk, kurtosis = ku, shapiro_p = sw_p))
}

# ---- 2. 自动寻找最佳转换（bestNormalize 包） ----
# 这个包会自动尝试多种转换，选最佳！
set.seed(42)
x_bad <- rlnorm(200, 0, 2)  # 严重偏态

bn <- bestNormalize(x_bad)
bn  # 列出所有尝试的转换及其效果
# 可能的输出：orderNorm transformation (ORQ) 通常最佳
# 备选：Box-Cox, Yeo-Johnson, sqrt, log, arcsinh

x_transformed <- predict(bn)
normality_diagnostic(x_bad, "原始数据")
normality_diagnostic(x_transformed, "转换后")

# ---- 3. Box-Cox 的稳健做法 ----
# 先 winsorize（把极端值拉到 1%/99% 分位数）
winsorize <- function(x, probs = c(0.01, 0.99)) {
  q <- quantile(x, probs, na.rm = TRUE)
  x[x < q[1]] <- q[1]
  x[x > q[2]] <- q[2]
  return(x)
}

x_clean <- winsorize(x_bad)
bc <- powerTransform(x_clean + 0.001)  # 小偏移避免 log(0)
x_bc <- bcPower(x_clean + 0.001, bc$lambda)
cat("Box-Cox λ =", round(bc$lambda, 3), "\n")

# ---- 4. 群落数据转换 —— 特别注意！ ----
# ❌ 错误：对群落数据做 Z-score
# ✅ 正确：Hellinger 转换
spe <- matrix(rpois(150, lambda = 5), nrow = 10)
spe_hel <- decostand(spe, method = "hellinger")

# 为什么不用 Z-score？
# → Z-score 让每个物种的均值变为 0
# → 破坏了物种之间的相对丰度关系
# → 双零（两个样方都没有物种A）被当作"相似"，但它们环境可能完全不同
```

## A3 — 数据标准化（群落数据专题）

### "双零问题"（The Double-Zero Problem）

这是数量生态学中最容易被忽视的核心问题。

```
场景：两个样方都没有物种A

如果用欧氏距离 + 原始丰度：
  距离 = 0（"完全一样"）
  但！一个样方是高山溪流，一个是平原湖泊 → 环境完全不同
  
→ 这就是"双零问题"：零丰度被错误地解释为"相似"
```

### 群落数据标准化方法选择

| 转换方法 | 解决双零？ | 保留丰度差？ | 适用于 | 不适用于 |
|:---------|:----------|:-----------|:------|:--------|
| **Hellinger** | ✅ 是 | 部分 | PCA/RDA/k-means | 需要保留绝对丰度 |
| **Chord** | ✅ 是 | 部分 | PCA/RDA（备选） | 小样方 |
| **Wisconsin** | ❌ 否 | ✅ 是 | 生态学经典 | PCA（一般不用Wisconsin+PCA） |
| **Chi-square** | ✅ 是 | ❌ 否 | CA/CCA | PCA/RDA |
| **Z-score** | ❌ 否（更惨）| ❌ 否 | ❌ 不要用在群落数据！ | — |
| **PA (presence/absence)** | ❌ 否 | ❌ 否 | 粗粒度分析 | 需要丰度信息 |

### 关键选择逻辑

```
你要做什么？                      →  转换方法
├─ PCA + 群落数据                → Hellinger 转换（必选！）
├─ RDA + 群落数据                → Hellinger 转换（必选！）
├─ NMDS + Bray-Curtis           → 不需要转换（Bray-Curtis 自带免疫）
├─ PERMANOVA + Bray-Curtis      → 不需要转换
├─ CA / CCA                     → Chi-square 距离（vegan 自动处理）
├─ 聚类 + 欧氏距离              → Hellinger 或 Chord
└─ α 多样性                    → 原始计数数据（不要转换！）
```

### 环境变量标准化

```
环境变量的标准化比较简单：
├─ 不同单位（pH + 温度 + 深度）
│   └─ → Z-score（decostand(method="standardize")）
├─ 同一单位
│   └─ → 可选不标准化（看相对重要性）
└─ 用于 PCA 或 RDA 时
    └─ → 始终标准化（否则变量尺度主导排序）
```

### R 代码

```r
# ============================================================
# A3 增强版：群落数据标准化的正确姿势
# ============================================================

# ---- 关键对比：转换前后 PCA 的差异 ----
# 模拟一个有双零问题的群落矩阵
set.seed(1)
spe_bad <- matrix(0, nrow = 20, ncol = 10)
# 前10个样方只有前5个物种
spe_bad[1:10, 1:5] <- matrix(rpois(50, 20), nrow = 10)
# 后10个样方只有后5个物种  
spe_bad[11:20, 6:10] <- matrix(rpois(50, 20), nrow = 10)
# → 前10个样方和后10个样方没有共享物种！
# → 原始丰度的欧氏距离会认为它们非常"相似"（大量双零）

# ❌ 错误做法：原始数据 + PCA
pca_bad <- rda(spe_bad)  # PCA 默认用欧氏距离
plot(pca_bad, main = "❌ 原始数据 PCA — 前10后10混在一起（假象！）")

# ✅ 正确做法：Hellinger + PCA
spe_hel <- decostand(spe_bad, method = "hellinger")
pca_good <- rda(spe_hel)
plot(pca_good, main = "✅ Hellinger转换后 PCA — 前10后10清晰分开")

# ---- 各种转换的快速对比 ----
compare_transformations <- function(spe) {
  methods <- c("hellinger", "chord", "total", "max", "pa", "wisconsin", "normalize")
  results <- list()
  
  for (m in methods) {
    if (m == "normalize") {
      transformed <- decostand(spe, method = "normalize")
      label <- "Chord"
    } else {
      transformed <- decostand(spe, method = m)
      label <- m
    }
    # 计算转换后矩阵的欧氏距离
    dist_euc <- dist(transformed)
    results[[label]] <- dist_euc
  }
  return(results)
}
```

## A4 — 缺失值处理（MCAR/MAR/MNAR 深度解析）

### 缺失机制的三种类型

```
MCAR (Missing Completely At Random)
├─ 缺失概率与任何变量（包括缺失变量自身）都无关
├─ 例：试管意外打翻 → 随机丢失
├─ 检测：Little's MCAR test
├─ 处理：删除 OK（如果 <5%），任何插补方法都 OK
└─ 现实中极少见！

MAR (Missing At Random) ← 最常见
├─ 缺失概率与其他已观测变量有关，与缺失变量本身无关
├─ 例：深水站点更难采到某些物种，但"难采"可由"深度"这个已观测变量解释
├─ 检测：无法直接检验，只能通过辅助变量推断
├─ 处理：MICE 是最佳选择
└─ ❌ 不能简单删除！

MNAR (Missing Not At Random) ← 最棘手
├─ 缺失概率与缺失变量本身的值有关
├─ 例：稀有物种更难被检测到 → 丰度越低越"缺失"
├─ 检测：通常无法从数据中检验
├─ 处理：需要专门模型（模式混合模型、选择模型）
└─ MICE 在这里会有偏倚！
```

### 方法选择矩阵

| 方法 | MCAR | MAR | MNAR | 缺失<5% | 5-20% | 20-50% | >50% |
|:-----|:----:|:---:|:----:|:-------:|:-----:|:------:|:----:|
| **删除** | ✅ | ❌ | ❌ | ✅ | ⚠️ | ❌ | ❌ |
| **均值填充** | ✅ | ⚠️ | ❌ | ✅ | ❌ | ❌ | ❌ |
| **KNN** | ✅ | ✅ | ⚠️ | ✅ | ✅ | ⚠️ | ❌ |
| **missForest** | ✅ | ✅ | ⚠️ | ✅ | ✅ | ✅ | ⚠️ |
| **MICE** | ✅ | ✅ | ❌ | ✅ | ✅ | ✅ | ⚠️ |
| **模式混合模型** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ⚠️ |

### 实战决策：你的缺失数据该怎么办？

```
Step 1: 评估缺失比例
├─ < 5% → 删除缺失行（listwise删除）OK → 完成
├─ 5-20% → 进入 Step 2
├─ 20-50% → 进入 Step 2，考虑 MICE 或 missForest
└─ > 50% → 认真考虑是否放弃该变量

Step 2: 评估缺失模式
├─ 完全随机 → 删除 OK
├─ 某变量缺失比率高 → 思考：为什么这些值缺失？
│   └─ 例：深水站点的 DO 总是缺失 → MAR（depth 可解释）
└─ 缺失变量本身决定缺失 → MNAR → 需要专门方法

Step 3: 选择插补方法
├─ MAR + 连续变量 + 变量间有合理相关性 → MICE (pmm 方法)
├─ MAR + 复杂数据（非线性+交互） → missForest
├─ MAR + 时间序列 → 线性插值 / Kalman 平滑
└─ MNAR → 报告偏倚方向 + 敏感性分析
```

```r
# ---- 1. 缺失值全局诊断 ----
missing_diagnostic <- function(df) {
  # 缺失比例
  miss_pct <- colMeans(is.na(df)) * 100
  cat("\n=== 缺失值诊断 ===\n")
  cat("总观测数:", nrow(df), "\n")
  cat("完全完整行:", sum(complete.cases(df)), 
      "(", round(sum(complete.cases(df))/nrow(df)*100, 1), "%)\n\n")
  
  cat("各变量缺失比例:\n")
  for (i in seq_along(miss_pct)) {
    flag <- if(miss_pct[i] > 20) "🚫" 
            else if(miss_pct[i] > 5) "⚠️" 
            else "✓"
    cat(sprintf("  %s %-15s: %5.1f%%\n", flag, names(miss_pct)[i], miss_pct[i]))
  }
  
  # 缺失模式
  cat("\n缺失模式 (前10种):\n")
  miss_pattern <- md.pattern(df, plot = FALSE)
  print(head(miss_pattern, min(10, nrow(miss_pattern))))
  
  # 可视化
  vis_miss(df, sort_miss = TRUE) + 
    labs(title = "缺失值分布热图")

# ---- 2. 比较不同插补方法的效果 ----
compare_imputation <- function(df_complete, miss_rate = 0.2, seed = 42) {
  set.seed(seed)
  
  # 制造缺失
  df_miss <- df_complete
  n_miss <- round(nrow(df_miss) * miss_rate)
  miss_idx <- sample(1:nrow(df_miss), n_miss)
  true_values <- df_miss[miss_idx, 1]
  df_miss[miss_idx, 1] <- NA
  
  # 方法1: 均值插补
  df_mean <- df_miss
  df_mean[miss_idx, 1] <- mean(df_miss[, 1], na.rm = TRUE)
  rmse_mean <- sqrt(mean((df_mean[miss_idx, 1] - true_values)^2))
  
  # 方法2: MICE
  imp <- mice(df_miss, m = 5, method = "pmm", printFlag = FALSE, seed = seed)
  df_mice <- complete(imp)
  rmse_mice <- sqrt(mean((df_mice[miss_idx, 1] - true_values)^2))
  
  # 方法3: missForest
  imp_mf <- missForest(df_miss, verbose = FALSE)
  df_mf <- imp_mf$ximp
  rmse_mf <- sqrt(mean((df_mf[miss_idx, 1] - true_values)^2))
  
  cat("\n=== 插补方法比较 (缺失", miss_rate*100, "%) ===\n")
  cat(sprintf("均值填充 RMSE:  %.3f\n", rmse_mean))
  cat(sprintf("MICE插补  RMSE:  %.3f\n", rmse_mice))
  cat(sprintf("missForest RMSE: %.3f\n", rmse_mf))
  cat("→ RMSE越小越好，但也要看不确定性！\n")
  cat("→ MICE 会给出多个插补版本(m>1)，反映插补不确定性\n")
}

  # 如果差异大 → 插补模型可能有问题
}
```

## A5 — EDA（探索性数据分析）增强版

### EDA 不是"看一眼 summary()"

完整 EDA 应包含：

1. **数据完整性检查** — 缺失、重复、异常
2. **分布检查** — 每个变量的分布特征
3. **关系检查** — 变量间的相关性、非线性关系
4. **结构检查** — 哪些变量应该被转换？哪些变量是冗余的？

### R 一站式 EDA 报告

```r

# 一行生成完整 HTML 报告

# ---- 手写 EDA 流水线 ----
eda_pipeline <- function(df, group_var = NULL) {
  cat("\n========== EDA Pipeline ==========\n")
  
  # 1. 数据结构
  cat("\n--- 数据结构 ---\n")
  cat("观测数:", nrow(df), " 变量数:", ncol(df), "\n")
  str(df, give.attr = FALSE)
  
  # 2. 缺失值
  cat("\n--- 缺失值 ---\n")
  miss <- colSums(is.na(df))
  if (sum(miss) > 0) {
    print(miss[miss > 0])
  } else {
    cat("✓ 无缺失值\n")
  }
  
  # 3. 数值变量摘要
  cat("\n--- 数值变量 ---\n")
  num_vars <- names(df)[sapply(df, is.numeric)]
  for (v in num_vars) {
    x <- df[[v]]
    cat(sprintf("%-20s mean=%.2f sd=%.2f min=%.2f max=%.2f skew=%.2f\n",
                v, mean(x, na.rm=TRUE), sd(x, na.rm=TRUE),
                min(x, na.rm=TRUE), max(x, na.rm=TRUE),
                moments::skewness(x, na.rm=TRUE)))
  }
  
  # 4. 高相关性对（|r| > 0.7）
  cat("\n--- 高相关性 (|r| > 0.7) ---\n")
  cor_mat <- cor(df[num_vars], use = "pairwise.complete.obs")
  high_cor <- which(abs(cor_mat) > 0.7 & upper.tri(cor_mat), arr.ind = TRUE)
  if (nrow(high_cor) > 0) {
    for (i in 1:nrow(high_cor)) {
      cat(sprintf("  %s ↔ %s: r = %.3f\n",
                  rownames(cor_mat)[high_cor[i,1]],
                  colnames(cor_mat)[high_cor[i,2]],
                  cor_mat[high_cor[i,1], high_cor[i,2]]))
    }
  } else {
    cat("✓ 无极高相关性\n")
  }
  
  # 5. 分组比较（如果有分组变量）
  if (!is.null(group_var) && group_var %in% names(df)) {
    cat("\n--- 分组比较 (", group_var, ") ---\n")
    group_summary <- df %>%
      group_by(!!sym(group_var)) %>%
      summarise(
        n = n(),
        across(where(is.numeric), 
               list(mean = ~mean(.x, na.rm=TRUE),
                    sd = ~sd(.x, na.rm=TRUE)),
               .names = "{.col}_{.fn}")
      )
    print(group_summary, n = 50)
  }
  
  cat("\n========== EDA 完成 ==========\n")
}
```

## 📋 A类完整诊断清单（打印使用）

```
在进入任何分析之前，逐项检查：

□ 1. 数据完整性
    □ 所有变量名一致（没有空格、特殊字符）
    □ 分类变量确认为 factor 类型
    □ 日期确认 Date 类型
    □ 没有完全重复的行

□ 2. 异常值
    □ 逐个连续变量做 boxplot 检查
    □ 偏态变量用 adjusted boxplot
    □ 发现异常值 → 核对原始记录 → 决定保留/修正/删除

□ 3. 分布检查
    □ 关键变量做 Q-Q 图 + 偏度检查
    □ 需要正态的变量检查 |skewness| < 1
    □ 计数数据检查过离散（方差/均值 > 1.5）

□ 4. 缺失值
    □ 计算每个变量的缺失比例
    □ 缺失 < 5% → 删除或简单填充 OK
    □ 缺失 5-50% → MICE 或 missForest
    □ 缺失 > 50% → 考虑删除该变量

□ 5. 共线性
    □ 计算 VIF（如后续做回归）
    □ |r| > 0.7 的变量对：保留生态学上"更直接"的那个
    □ 多重共线性 → 考虑 PCA 降维或 Ridge 回归

□ 6. 群落数据特殊检查
    □ 确认是否对群落数据做了 Z-score（❌ 不应该！）
    □ PCA 前确认用了 Hellinger/Chord 转换
    □ 稀有物种是否过滤（< 3次出现可以考虑去除）

□ 7. 实验设计检查
    □ 确认了嵌套/重复测量结构
    □ 空间坐标是否已确认（用于后续 D 类分析）
    □ 时间序列的时间间隔是否均匀
```
