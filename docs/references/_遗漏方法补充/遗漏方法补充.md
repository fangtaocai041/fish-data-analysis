## 1. RLQ 分析 + Fourth-Corner（性状-环境-群落三角）🆕

**来源**: Numerical Ecology with R 第7章 · Dray & Legendre 2008

### 做什么？
连接三张表：**物种-样方矩阵(L)** × **环境变量(R)** × **物种性状(Q)**，一次性回答"哪些性状使物种适应哪些环境？"

### 与现有方法的关系
- 现有 B4（功能多样性）只算指数 → RLQ 告诉你**为什么**功能多样性变化
- 现有 RDA 只建模物种-环境 → RLQ 通过性状**解释**物种响应

### R 代码

```r
library(ade4)

# 三张表
# spe: 物种×样方 (L表)
# env: 样方×环境 (R表)  
# traits: 物种×性状 (Q表)

# Step 1: 分别分析
pca_env <- dudi.pca(env, scannf = FALSE)        # 环境PCA
coa_spe <- dudi.coa(spe, scannf = FALSE)         # 群落对应分析
pca_trait <- dudi.pca(traits, scannf = FALSE)    # 性状PCA（需先标准化）

# Step 2: RLQ分析（核心！）
rlq_out <- rlq(pca_env, coa_spe, pca_trait, scannf = FALSE)
summary(rlq_out)

# 可视化
plot(rlq_out)  # 同时展示物种、环境、性状在共享空间中的位置

# Step 3: Fourth-Corner 显著性检验
fc <- fourthcorner(env, spe, traits, modeltype = 6, 
                    p.adjust.method.G = "holm", nrepet = 999)
plot(fc)  # 热图：哪些性状-环境组合显著相关
```

### 适用边界

| 维度 | 说明 |
|:-----|:-----|
| **适用** | 需要同时解释"性状×环境×物种"三角关系 |
| **不适用** | 三张表各自独立时（分开分析即可） |
| **数据要求** | 环境变量和性状最好 ≤ 10个，太多导致解释困难 |
| **稳健性** | ⭐⭐⭐⭐ 置换检验保证显著性可靠；Type I error 控制需注意 |

## 2. 占用模型 (Occupancy Models) 🆕

**来源**: MacKenzie et al. 2002 · Bayesian Data Analysis in Ecology

### 做什么？
估计物种的**真实分布概率 (ψ)**，同时校正**探测概率 (p < 1)**。这是解决"没看到 ≠ 不存在"问题的标准方法。

### 为什么现有知识库缺失？
我们现有的 SDM (MaxEnt/GLM) 假设"不存在 = 真不存在 + 没探测到"，但忽略了探测概率。占用模型显式建模这两个过程。

### R 代码

```r
library(unmarked)

# 数据格式：每样方多次重复调查
# y: 样方×重复调查的探测/未探测矩阵 (0/1)
# siteCovs: 样方协变量（影响占用概率 ψ）
# obsCovs: 观测协变量（影响探测概率 p）

# 单季节占用模型
occ_model <- occu(~ obs_cov ~ site_cov, data = umf)
#                 p公式    ψ公式

summary(occ_model)

# 预测占用概率（校正了探测概率！）
psi_hat <- predict(occ_model, type = "state")  # ψ
p_hat   <- predict(occ_model, type = "det")    # p

# 多季节占用模型（动态——殖民和灭绝）
occ_dyn <- colext(psiformula = ~1,      # 初始占用
                  gammaformula = ~1,     # 殖民概率
                  epsilonformula = ~1,   # 灭绝概率
                  pformula = ~1,         # 探测概率
                  data = umf)
```

### 适用边界

| 维度 | 说明 |
|:-----|:-----|
| **适用** | 任何"可能没探测到"的调查数据（鸟类、鱼类、两栖类） |
| **不适用** | 探测概率 ≈ 1 的调查（如实验室数据） |
| **数据要求** | 每样方 ≥ 2 次重复调查；样方数 ≥ 30 |
| **替代方案** | N-mixture模型（丰度而非存在/不存在）；单次调查→SDM |

## 3. DHARMa 残差诊断（GLM/GLMM 残差黄金标准）🆕

**来源**: Hartig 2016 · 已取代传统残差图

### 做什么？
传统 GLM/GLMM 残差（Pearson/Deviance）不符合正态 → QQ图没用。DHARMa 用**模拟残差**，将任何 GLM/GLMM 的残差转化为统一尺度。

### R 代码

```r
library(DHARMa)

# 拟合模型
mod <- glmmTMB(count ~ treatment + (1|site), 
               family = nbinom2, data = df)

# DHARMa 残差诊断
sim_res <- simulateResiduals(mod, n = 1000)

# 一句命令出全套诊断图
plot(sim_res)
# → QQ图：均匀分布检验
# → 残差 vs 拟合：检查异方差
# → 可加：残差 vs 每个预测变量

# 专项检验
testDispersion(sim_res)       # 过离散
testZeroInflation(sim_res)    # 零膨胀
testSpatialAutocorrelation(sim_res, x, y)  # 空间自相关
testTemporalAutocorrelation(sim_res, time) # 时间自相关
```

### 为什么替代传统方法？
传统残差图的 QQ-plot 对 GLM/GLMM 不适用 → DHARMa 让你像检查 LM 残差一样检查 GLMM 残差。

## 4. piecewiseSEM（生态学结构方程模型）🆕

**来源**: Lefcheck 2016 · Methods in Ecology and Evolution

### 做什么？
将多个回归关系拼接成**因果网络**。与 `lavaan` 不同，piecewiseSEM 是为生态学设计的——允许小样本、非正态、混合效应、系统发育校正。

### R 代码

```r
library(piecewiseSEM)

# 假设因果模型：
# 干扰 → 生境异质性 → 物种多样性 → 功能多样性

# 分别拟合每个"组件模型"（可以用任何R建模函数！）
m1 <- lm(habitat ~ disturbance, data = df)
m2 <- glmer(richness ~ habitat + disturbance + (1|site), 
            family = poisson, data = df)
m3 <- lm(FDis ~ richness + habitat, data = df)

# 拼成 SEM
sem_model <- psem(m1, m2, m3)

# 诊断
summary(sem_model)
# → Fisher's C：整体模型拟合（p > 0.05 为接受）
# → 各路径的标准化系数和 p 值
# → d-separation tests：检查遗漏路径

# 如果 Fisher's C 显著（模型拟合差）
# → 查看 missing paths → 加回去
```

### 适用边界

| 维度 | 说明 |
|:-----|:-----|
| **适用** | 有因果假设的多变量系统 |
| **不适用** | 纯探索、没有因果假设（用多元回归即可） |
| **数据要求** | 每个路径至少10-20观测/参数 |
| **关键限制** | DAG 必须预先指定——不能从数据中"发现"因果 |

## 5. 模型平均 / 多模型推断 (MuMIn) 🆕

**来源**: Burnham & Anderson 2002 · Zuur 系列推荐

### 做什么？
当你不确定哪个模型"最好"时，不选一个，而是**对所有候选模型做加权平均**。AIC 权重反映各模型的证据强度。

### R 代码

```r
library(MuMIn)

# 全局模型
global_mod <- glm(count ~ pH * temp + DO + depth + NO3 + I(temp^2),
                  family = poisson, data = df, na.action = "na.fail")

# 自动化：所有子集组合 (dredge)
all_models <- dredge(global_mod, rank = "AICc")

# 查看结果
head(all_models)

# ΔAICc < 2 的模型取平均
avg_model <- model.avg(all_models, subset = delta < 2)
summary(avg_model)
# → 每个变量的"平均效应" + "相对重要性"（出现在多少模型中）

# ⚠️ 注意：dredge 是计算密集型，变量多时很慢！
# 建议：先用生态学知识预筛选变量，不要 dump 所有变量进去
```

### 为什么要模型平均？
单一"最优模型"可能只是数据中噪音的最优 → 模型平均承认不确定性 → 更稳健的推断。

## 6. 正则化方法 (LASSO / Ridge / Elastic Net) 🆕

**来源**: glmnet · 在 I 类 README 中被提及但无详解

### 做什么？
当变量数 > 样本数（p > n）或共线性严重时，正则化**收缩系数**防止过拟合。

| 方法 | 惩罚 | 效果 | 适用 |
|:-----|:-----|:-----|:-----|
| **Ridge** | λ Σβ² | 收缩但不归零 | 所有变量都可能重要 |
| **LASSO** | λ Σ|β| | 收缩到零 → 变量选择 | 大部分变量不重要 |
| **Elastic Net** | 混合两者 | 折中 | 不确定时 |

### R 代码

```r
library(glmnet)

# 准备矩阵
x <- model.matrix(~ pH + temp + DO + depth + NO3 - 1, data = df)
y <- df$species_richness

# LASSO（α = 1）
lasso_cv <- cv.glmnet(x, y, alpha = 1, family = "gaussian")
plot(lasso_cv)

# 最优 λ（最小 MSE，或 1-SE 规则）
best_lambda <- lasso_cv$lambda.min
coef(lasso_cv, s = best_lambda)  # 非零系数 = 被选中的变量

# Elastic Net（α = 0.5）
enet_cv <- cv.glmnet(x, y, alpha = 0.5)
```

```r
# ============================================================
# MaxEnt 完整调参流程（ENMeval）
# ============================================================
library(ENMeval)
library(dismo)

# 环境栅格
env_stack <- stack(list.files("climate/", pattern = ".tif$", full.names = TRUE))

# 存在点
occs <- read.csv("occurrences.csv")

# ENMeval 调参（block 空间分块CV，防止空间自相关导致过拟合）
eval_results <- ENMevaluate(
  occs = occs[, c("longitude", "latitude")],
  envs = env_stack,
  bg.coords = background_points,
  algorithm = "maxnet",
  tune.args = list(
    fc = c("L", "LQ", "H", "LQH", "LQHP"),       # feature classes
    rm = c(0.5, 1, 2, 3, 5)                       # regularization
  ),
  partitions = "block",    # ⚡ 关键：空间分块而非随机！
  partition.settings = list(orientation = "lat_lon"),
  parallel = TRUE
)

# 选最优模型：ΔAICc < 2 的都可以
eval_results@results %>% 
  filter(delta.AICc < 2) %>%
  arrange(AICc)

# 最终模型（选 ΔAICc 最小的）
best <- eval_results@results[1, ]
final_mod <- eval_results@models[[best$tune.args]]

# 预测
pred <- predict(final_mod, env_stack, type = "cloglog")
plot(pred)

# ============================================================
# 生态位等价性/相似性检验（ecospat）
# ============================================================
library(ecospat)

# 准备两个物种的存在点 + 背景环境
# 提取环境PCA空间
pca_env <- dudi.pca(na.omit(values(env_stack)), scannf = FALSE, nf = 2)
scores_glob <- pca_env$li

# 物种A和B的环境得分
spA_scores <- suprow(pca_env, extract(env_stack, occs_A))$li
spB_scores <- suprow(pca_env, extract(env_stack, occs_B))$li

# 计算生态位重叠

  z1 = spA_grid, z2 = spB_grid, 
  method = "D"  # Schoener's D
)

# 等价性检验
eq_test <- ecospat.niche.equivalency.test(
  z1 = spA_grid, z2 = spB_grid,
  rep = 999
)

# 相似性检验
sim_test <- ecospat.niche.similarity.test(
  z1 = spA_grid, z2 = spB_grid,
  rep = 999, alternative = "greater"
```

## 8. F 类补充：完整网络分析代码 🆕

**F-深度详解零代码**，这是最迫切需要补充的。

```r
# ============================================================
# 食物网 + 共现网络 + 二分网络 完整代码
# ============================================================
library(igraph)
library(bipartite)
library(SpiecEasi)  # 成分数据共现网络（如果需要）
library(vegan)

# ---- 1. 食物网拓扑指标 ----
foodweb <- graph_from_adjacency_matrix(mat, mode = "directed")

# 基本指标
vcount(foodweb)     # 物种数 S
ecount(foodweb)     # 连接数 L
edge_density(foodweb)  # Connectance = L / (S*(S-1))
diameter(foodweb)      # 最长食物链
mean_distance(foodweb) # 平均食物链长度

# 节点重要性
degree(foodweb, mode = "in")    # 入度（猎物数）
degree(foodweb, mode = "out")   # 出度（捕食者数）
betweenness(foodweb)             # 介数中心性（关键节点）

# 模块性
comm <- cluster_fast_greedy(as.undirected(foodweb))
modularity(comm)

# ---- 2. 二分网络（互作网络） ----
# 矩阵：行=植物 列=传粉者
bip_metrics <- networklevel(bip_mat, 
  index = c("connectance", "nestedness", "H2", "NODF"))

# 嵌套性（NODF）
nodf <- nested(bip_mat, method = "NODF")

# 零模型检验嵌套性是否显著
nulls <- nullmodel(bip_mat, N = 999, method = "r2d")
null_nodf <- sapply(nulls, function(x) nested(x, method = "NODF"))
p_value <- (sum(null_nodf >= nodf) + 1) / 1000

# 模块性
modules <- computeModules(bip_mat)
plotModuleWeb(modules)  # 可视化模块

# 物种级别特化度 d'
species_level <- specieslevel(bip_mat, index = "d")

# ---- 3. 共现网络（从群落数据构建） ----
# 注意：共现 ≠ 互作！
cor_mat <- cor(spe, method = "spearman")

# 只保留显著且足够强的相关
library(Hmisc)
cor_test <- rcorr(as.matrix(spe), type = "spearman")
sig_mat <- cor_test$P < 0.05
cor_mat[!sig_mat] <- 0
cor_mat[abs(cor_mat) < 0.3] <- 0  # 弱相关过滤

# 构建igraph
g_cooccur <- graph_from_adjacency_matrix(cor_mat, 
  mode = "undirected", weighted = TRUE, diag = FALSE)

# 正负边分开着色
E(g_cooccur)$color <- ifelse(E(g_cooccur)$weight > 0, "blue", "red")

# 可视化
plot(g_cooccur, 
     vertex.size = log(colSums(spe)) * 2,
     vertex.label.cex = 0.7,
     edge.width = abs(E(g_cooccur)$weight) * 2)
```
