## B1 — α 多样性（完整方法论）

### 一、多样性指数的"有效种数"框架（Hill Numbers）

**核心洞见**：不同指数衡量的实际上是不同"视角"的物种数。

| Hill 阶数 q | 指数 | 生态学含义 | 对稀有物种的权重 |
|:-----------|:-----|:-----------|:---------------|
| q = 0 | 物种丰富度 (S) | 只看"有没有" | 所有物种等权（1个个体=1000个个体） |
| q = 1 | exp(H') — 有效 Shannon | 典型个体来自哪个物种 | 按丰度比例加权 |
| q = 2 | Inverse Simpson (1/D) | 优势个体的有效种数 | 强烈偏好优势种 |

```
为什么 Hill Numbers 是更好的报告方式？
→ Shannon = 3.2 → 是什么意思？难以直观理解
→ exp(3.2) = 24.5 → "等价的物种数" ← 更容易理解！
→ 报告 exp(H') 而非 H' → 审稿人和读者都感谢你
```

### 二、Shannon vs Simpson：到底该报告哪个？

| 场景 | 推荐 | 理由 |
|:-----|:-----|:-----|
| 完整的群落调查 | **两者都报告** + Hill q=1 和 q=2 | 最完整 |
| 关注稀有物种 | 偏好 Shannon | 对稀有物种敏感 |
| 关注群落优势度 | 偏好 Simpson | 对优势度敏感 |
| 受采样不完整影响 | **Simpson 更稳健** | 少捕几个稀有种，Simpson 变化小 |
| 用于统计检验 | Simpson（在方差齐性方面更好） | Shannon 的方差在低丰富度时不稳 |
| 发表论文 | exp(H') 和 1/D（有效种数） | 直观可比 |

### 三、采样不完整：最被忽视的问题

**核心问题**：几乎所有 α 多样性指数都假设你看到了所有物种。但实际上你没有。

#### 稀疏曲线（Rarefaction Curve）解读

```
稀疏曲线达到渐近线？
├─ ✅ 已接近渐近线 → 采样充分 → 可以直接比较原始指数
├─ ⚠️ 仍在上升 → 采样不充分 → 必须用稀疏化到统一样本量再比较
└─ 🚫 远未达到渐近线 → 多样性指数不可靠 → 用 Chao1 估计
```

#### 稀疏化 vs 外推（iNEXT 框架）

```r
# ❌ 错误做法：直接比较不同样本量的 Shannon
site_A: n=1000个体, Shannon=3.5
site_B: n=200个体,  Shannon=2.8
→ 不能直接说 A 的多样性更高！样本量不同！

# ✅ 正确做法：稀疏化到相同覆盖度（coverage-based rarefaction）
library(iNEXT)
# iNEXT 基于覆盖度（而非样本量）标准化
# 覆盖度 = 1 - (singleton数/总个体数)
```

### 四、Chao1 的陷阱

```
Chao1 公式: S_est = S_obs + (F1² / 2F2)

其中 F1 = singleton 数量, F2 = doubleton 数量

关键限制：
1. 如果 F1 = 0（所有物种都被捕获≥2次）→ Chao1 = S_obs（无修正）
2. 如果 F2 = 0（所有稀有种都是singleton）→ Chao1 → ∞（无意义！）
3. Chao1 是下界估计，不是无偏估计 → 报告置信区间！
4. 不要比较不同样本量的 Chao1 → 用 iNEXT 的渐近估计
```

### 五、α 多样性的统计检验

| 你的设计 | 检验方法 | R 代码 |
|:---------|:---------|:------|
| 两组比较 | Welch t-test 或 Mann-Whitney | `t.test()` / `wilcox.test()` |
| 多组比较 | One-way ANOVA 或 Kruskal-Wallis | `aov()` / `kruskal.test()` |
| 有协变量 | ANCOVA 或 GLM | `lm(Shannon ~ group + covariate)` |
| 嵌套设计 | LMM | `lmer(Shannon ~ group + (1|block))` |

**注意**：Shannon 的分布通常接近正态（n>10时），Simpson 偏态需谨慎。

### 六、α 多样性诊断代码

```r
# ============================================================
# B1 增强版：α多样性完整诊断
# ============================================================
library(iNEXT)      # 基于覆盖度的稀疏/外推
library(hillR)      # Hill numbers 的简单接口
library(vegan)

# ---- 1. Hill Numbers 一键计算 ----
alpha_hill <- data.frame(
  site = rownames(spe_mat),
  q0 = hillR::hill_taxa(spe_mat, q = 0),  # 物种丰富度
  q1 = hillR::hill_taxa(spe_mat, q = 1),  # exp(Shannon)
  q2 = hillR::hill_taxa(spe_mat, q = 2)   # Inverse Simpson
)
# q1/q0 = 均匀度比率（如果接近1，群落非常均匀）
alpha_hill$evenness_q1 <- alpha_hill$q1 / alpha_hill$q0

# ---- 2. 采样充分性诊断 ----
# iNEXT: 基于覆盖度的稀疏化和外推
inext_out <- iNEXT(spe_mat, q = c(0, 1, 2), datatype = "abundance")
# 关键输出：每个样方的"样本覆盖度"（sample coverage）
# coverage > 0.95 → 采样充分
# coverage 0.85-0.95 → 勉强，用稀疏化比较
# coverage < 0.85 → 必须用估计方法（非原始指数）

# 可视化
ggiNEXT(inext_out, type = 1) +  # 稀疏曲线
  labs(title = "基于覆盖度的稀疏化与外推")

# ---- 3. 稀疏化到统一样本量再比较 ----
# 找出最小样本量和统一覆盖度
min_n <- min(rowSums(spe_mat))
# 稀疏化到最小样本量
spe_rar <- rrarefy(spe_mat, sample = min_n)
alpha_rar <- data.frame(
  site = rownames(spe_rar),
  Shannon_rar = diversity(spe_rar, "shannon"),
  Simpson_rar = diversity(spe_rar, "simpson")
)

# ---- 4. Chao1 的置信区间 ----
chao_out <- estimateR(spe_mat)
# Chao1 的 SE 可以用来计算 95% CI
# 如果 CI 很宽（宽度 > Chao1 估计的 50%）→ 不可靠

# ---- 5. α 多样性与环境因子的回归 ----
# 如果 Shannon 不符合正态（n<10时常见），用 GLM
alpha_env <- merge(alpha_hill, env_data, by = "site")
summary(lm(q1 ~ pH + DO + temp, data = alpha_env))
```

## B2 — β 多样性（深度解析）

### 一、Bray-Curtis 为什么是"默认选择"？

```
Bray-Curtis = 1 - (2 * sum(min(abundance)) / sum(total_abundance))

三个关键特性：
1. 对"双零"免疫 → 两个样方都不含物种A ≠ 它们相似
2. 丰度加权 → 优势物种贡献更大
3. 不满足三角不等式 → 不能用某些方法（但 NMDS 和 PERMANOVA OK）
```

### 二、β 多样性分解：Baselga vs Podani

这是目前 β 多样性领域最大的争议。

| | Baselga (2010) | Podani (2011) |
|:--|:--------------|:------------|
| **核心思想** | β_total = turnover + nestedness | β_total = replacement + richness_diff |
| **Nestedness 怎么算** | 物种数少的样方是完全嵌套在多的里面 | 两个样方只是物种数不同 |
| **适用场景** | 岛屿生物地理学、明确嵌套结构 | 一般群落比较 |
| **争议** | 可能高估 nestedness 组分 | 更保守但公式更复杂 |
| **R 包** | `betapart` | `adespatial` (beta.div) |

```r
# 实际建议：两种都跑，如果结论一致 → 任一都行
# 如果结论不一致 → 在论文中报告两者的差异，并讨论原因
```

### 三、betadisper 的重要性（不能跳过！）

```
PERMANOVA 的 p < 0.05 可能是两种完全不同的情况：

情况1：组间均值真的不同（你想要的）
  → 组内离散度相同，组质心不同

情况2：组内离散度不同（你不想要的）
  → 组质心可能相同，但一组波动大另一组稳定
  → betadisper p < 0.05 → PERMANOVA 显著可能是假象！

所以：
  betadisper p > 0.05 → PERMANOVA 结果可信
  betadisper p < 0.05 → 
    需要同时报告位置差异 AND 离散度差异
    或者用 PERMDISP2 替代 PERMANOVA
```

### 四、β 多样性诊断代码

```r
# ============================================================
# B2 增强版：β多样性完整诊断
# ============================================================
library(betapart)
library(adespatial)  # beta.div (Podani 分解)
library(vegan)

# ---- 1. β 多样性相异矩阵选择指南 ----
compare_beta_indices <- function(spe) {
  indices <- list(
    "Bray-Curtis" = vegdist(spe, "bray"),
    "Jaccard"     = vegdist(decostand(spe, "pa"), "jaccard"),
    "Sørensen"    = vegdist(decostand(spe, "pa"), method = "bray", binary = TRUE),
    "Ružička"     = vegdist(spe, "ruzicka")
  )
  
  # 各矩阵之间的 Mantel 相关
  cor_matrix <- matrix(NA, 4, 4, 
                       dimnames = list(names(indices), names(indices)))
  for (i in 1:4) {
    for (j in 1:4) {
      cor_matrix[i,j] <- cor(c(indices[[i]]), c(indices[[j]]))
    }
  }
  # 如果相关性 > 0.9，选择不那么重要
  # 如果相关性 < 0.7，选择对结论有影响 → 报告敏感性
  return(cor_matrix)
}

# ---- 2. Baselga 分解 (betapart) ----
# β_total = β_turn + β_nest
beta_baselga <- beta.multi(spe_mat, index.family = "sorensen")
cat("Baselga 分解:\n")
cat("  总β多样性:", round(beta_baselga$beta.SOR, 3), "\n")
cat("  周转组分:", round(beta_baselga$beta.SIM, 3), 
    sprintf("(%.1f%%)", beta_baselga$beta.SIM/beta_baselga$beta.SOR*100), "\n")
cat("  嵌套组分:", round(beta_baselga$beta.SNE, 3), 
    sprintf("(%.1f%%)", beta_baselga$beta.SNE/beta_baselga$beta.SOR*100), "\n")

# ---- 3. Podani 分解 (adespatial) ----
beta_podani <- beta.div(spe_mat, method = "hellinger")
cat("\nPodani 分解:\n")
cat("  总β多样性:", round(beta_podani$beta$BDtotal, 3), "\n")
cat("  Replacement:", round(beta_podani$beta$Repl, 3), "\n")
cat("  RichnessDiff:", round(beta_podani$beta$RichDiff, 3), "\n")

# ---- 4. betadisper 完整诊断 ----
bc_dist <- vegdist(spe_mat, "bray")
bd <- betadisper(bc_dist, group)

# 不仅仅是 p 值！
cat("\n=== betadisper 完整诊断 ===\n")
cat("ANOVA p =", round(anova(bd)$`Pr(>F)`[1], 4), "\n")
cat("各组到质心的平均距离:\n")
print(tapply(bd$distances, bd$group, mean))
cat("各组到质心距离的 SD:\n")
print(tapply(bd$distances, bd$group, sd))

# 可视化：箱线图 + PCoA
boxplot(bd, main = "各组到质心的距离 — betadisper")
plot(bd)  # PCoA 图中显示质心和离散度

# 如果是显著 → 可能的原因
if (anova(bd)$`Pr(>F)`[1] < 0.05) {
  cat("\n⚠️ betadisper 显著！PERMANOVA 显著可能是离散度差异\n")
  cat("建议：同时报告位置差异（PERMANOVA）和离散度差异（betadisper）\n")
  cat("或者使用 PERMDISP2 进行更严格的检验\n")
}

# ---- 5. 距离衰减分析 ----
```

## B4 — 功能多样性（五种指数的完整对比）

### 为什么需要多个指数？

单看 FRic 或 Rao's Q 都只能看到功能多样性的一个侧面。五个指数共同构成了完整图景。

### 五指数对比表

| 指数 | 衡量什么 | 需要性状标准化？ | 最小物种数 | 对异常值敏感性 | 适合 |
|:-----|:---------|:----------------|:----------|:-------------|:-----|
| **FRic** | 功能空间体积 | ✅ 必须 | > T+1 | 🔴 极高 | 大群落 |
| **FEve** | 丰度在功能空间的均匀度 | ✅ 建议 | > 3 | 🟡 中等 | 任何群落 |
| **FDiv** | 丰度向边缘还是中心聚集 | ✅ 建议 | > 3 | 🟡 中等 | 任何群落 |
| **FDis** | 平均功能差异 | ✅ 建议 | ≥ 2 | 🟢 低 | **小群落首选** |
| **Rao's Q** | 随机两物种的功能差异期望 | ✅ 建议 | ≥ 2 | 🟢 低 | 配合 FDis |

### FRic 的关键限制

```
FRic (凸包体积) 的限制：
1. 物种数 S 必须 > 性状数 T + 1
   → 如果 S=5, T=4 → 凸包退化为3D平面，体积=0 → FRic=0（但群落可能功能多样！）
2. 极端性状值会极大膨胀凸包 → 必须先检查异常值
3. 性状必须标准化 → 否则尺度最大的变量主导

FRic 不可靠时的替代：用 FDis（功能离散度）
  → FDis 不需要 S > T+1
  → 对异常值也更稳健
```

### 功能性状类型与距离选择

| 性状类型 | 距离方法 | R 代码 |
|:---------|:---------|:------|
| 全部连续 | Gower 距离或欧氏距离 | `FD::gowdis()` 或 `dist()` |
| 混合（连续+分类+有序） | **Gower 距离**（唯一选择） | `FD::gowdis()` |
| 全部连续 + 无缺失 | 欧氏距离（更简单） | `dist(scale(traits))` |

### 功能多样性诊断代码

```r
# ============================================================
# B4 增强版：功能多样性完整诊断
# ============================================================
library(FD)
library(fundiversity)  # 更快的新包（推荐用于大矩阵）

# ---- 1. 性状数据质量检查 ----
check_traits <- function(traits) {
  cat("\n=== 性状数据质量 ===\n")
  cat("物种数:", nrow(traits), "\n")
  cat("性状数:", ncol(traits), "\n")
  cat("可计算 FRic? ", if(nrow(traits) > ncol(traits)+1) "✅ 是" else "❌ 否（S ≤ T+1）\n")
  
  # 检查缺失
  miss <- colSums(is.na(traits))
  if (sum(miss) > 0) {
    cat("⚠️ 有缺失性状:\n")
    print(miss[miss > 0])
    cat("建议：用 missForest 或 MICE 插补，或删除缺失>50%的性状\n")
  }
  
  # 检查极端值
  for (i in 1:ncol(traits)) {
    if (is.numeric(traits[[i]])) {
      x <- traits[[i]]
      z <- abs((x - mean(x, na.rm=TRUE)) / sd(x, na.rm=TRUE))
      if (any(z > 3, na.rm=TRUE)) {
        cat("⚠️ 性状", names(traits)[i], "有极端值 (|z| > 3):\n")
        print(which(z > 3))
      }
    }
  }
}

# ---- 2. 使用 fundiversity（更快） ----
# 对于有 S < T+1 限制的群落，分别计算
fd_indices_list <- list()
for (i in 1:nrow(spe_mat)) {
  # 取该样方中出现的物种
  present <- which(spe_mat[i, ] > 0)
  if (length(present) > ncol(traits_scaled) + 1) {
    fd_indices_list[[i]] <- fd_fric(traits_scaled[present, ])
  } else {
    fd_indices_list[[i]] <- data.frame(FRic = NA, note = "S ≤ T+1")
  }
}

# ---- 3. 功能多样性指数之间的相关性 ----
# 如果 FRic 和 FDis 高度相关(r > 0.8)，只需报告一个
fd_cor <- cor(fd_indices[, c("FRic", "FEve", "FDiv", "FDis", "RaoQ")],
              use = "complete.obs")
corrplot(fd_cor, method = "number", type = "upper",
         title = "功能多样性指数间的相关性")

# ---- 4. 功能多样性 vs 物种多样性 ----
# 关键检验：功能和物种多样性是否解耦？
alpha_fd <- merge(alpha_hill, fd_indices, by = "site")
with(alpha_fd, {
  cat("\nFRic vs Species Richness: r =", round(cor(q0, FRic, use="complete.obs"), 3), "\n")
  cat("RaoQ vs Simpson: r =", round(cor(q2, RaoQ, use="c"), 3), "\n")
})
# 如果 r > 0.9 → 功能多样性几乎被物种数完全解释 → 功能冗余高
# 如果 r < 0.5 → 功能和物种多样性"解耦" → 更有趣的生态学故事
```

## B5 — 系统发育多样性（零模型是关键）

### ses.MPD 和 ses.MNTD 的区别

| | MPD | MNTD |
|:--|:----|:-----|
| **全称** | Mean Phylogenetic Distance | Mean Nearest Taxon Distance |
| **计算** | 所有物种对的平均系统发育距离 | 每个物种到最近亲缘种的平均距离 |
| **生态学尺度** | 群落整体（深层进化） | 最近亲缘种（浅层/终端） |
| **NRI** | (MPD_obs - MPD_null) / SD_null | — |
| **NTI** | — | (MNTD_obs - MNTD_null) / SD_null |
| **正值含义** | 系统发育聚集（环境筛选） | 近缘种聚集 |
| **负值含义** | 系统发育发散（竞争排斥） | 近缘种排斥 |

### 零模型选择：这是最关键的决策

```
不同的 null model 可以得出完全相反的结论！

null.model = "taxa.labels"（最常用）
  → 重新随机分配物种名到树上
  → 保留：树拓扑 + 物种的丰富度/丰度
  → 破坏：物种的进化关系
  → 缺点：比较保守（p 值偏高）

null.model = "richness"
  → 从整个物种库中随机抽取（保持样方丰富度）
  → 保留：样方物种数
  → 破坏：物种身份
  → 更宽松，但也更不"现实"

null.model = "frequency"
  → 按出现频率随机抽取
  → 比 richness 更保留数据结构

null.model = "independentswap"
  → 保留行和 + 列和
  → 最保守

建议：至少报告两种零模型的结果，讨论差异。
```

### R 诊断代码

```r
# ============================================================
# B5 增强版：系统发育多样性完整诊断
# ============================================================
library(picante)
library(ape)

# ---- 1. 系统发育树的"质量检查" ----
check_phy <- function(phy, spe_mat) {
  cat("\n=== 系统发育树检查 ===\n")
  cat("是否超度量树？", if(is.ultrametric(phy)) "✅" else "❌ 需要分子钟校准", "\n")
  cat("是否二叉？", if(is.binary(phy)) "✅" else "⚠️ 有多歧分支", "\n")
  
  # 物种对应
  shared <- intersect(phy$tip.label, colnames(spe_mat))
  missing_tree <- setdiff(colnames(spe_mat), phy$tip.label)
  missing_data <- setdiff(phy$tip.label, colnames(spe_mat))
  
  if (length(missing_tree) > 0) {
    cat("⚠️ ", length(missing_tree), "个物种有数据但不在树上\n")
    cat("  建议从分析中移除或添加到树上\n")
  }
  cat("匹配物种:", length(shared), "/", ncol(spe_mat), "\n")
}

# ---- 2. 多个零模型的对比 ----
compare_null_models <- function(spe_mat, phy, runs = 999) {
  null_models <- c("taxa.labels", "richness", "frequency", "independentswap")
  results <- list()
  
  for (nm in null_models) {
    # 有些零模型需要距离矩阵
    if (nm %in% c("taxa.labels", "richness", "frequency")) {
      tryCatch({
        ses <- ses.mpd(spe_mat, cophenetic(phy), 
                       null.model = nm, runs = runs,
                       abundance.weighted = TRUE)
        results[[nm]] <- ses
      }, error = function(e) {
        cat("  ", nm, ": 不可用\n")
      })
    }
  }
  
  # 比较不同零模型的 ses 值
  cat("\n=== 零模型对比 ===\n")
  for (nm in names(results)) {
    ses <- results[[nm]]
    cat(sprintf("%-20s: ses.MPD = %+.3f, p = %.3f\n",
                nm, mean(ses$mpd.obs.z, na.rm=TRUE),
                mean(ses$mpd.obs.p, na.rm=TRUE)))
  }
  
  return(results)
}

# ---- 3. NRI/NTI 联合诊断 ----
# NRI > 0 + NTI > 0 → 环境筛选（近缘种和远缘种都聚集）
# NRI < 0 + NTI > 0 → 终端聚集但深层发散（复杂格局）
# NRI > 0 + NTI ≈ 0 → 深层聚类
# NRI < 0 + NTI < 0 → 竞争排斥主导
```

## 📋 B类完整诊断清单

```
□ 1. 采样充分性
    □ 所有样方跑 iNEXT 覆盖度 > 0.85
    □ 覆盖度不足的样方用稀疏化比较
    □ 稀疏曲线是否接近渐近线

□ 2. α 多样性报告
    □ 不要只报告 Shannon！至少 + Simpson
    □ 推荐报告 Hill numbers: q0, q1, q2
    □ 如果样本量不同 → 用 coverage-based rarefaction

□ 3. Chao1 / ACE
    □ 检查 singleton 和 doubleton 数量
    □ 如果 F1=0 或 F2=0 → Chao1 不可靠
    □ 报告 95% 置信区间

□ 4. β 多样性
    □ Bray-Curtis 是你的朋友（默认即可）
    □ 做了 PERMANOVA → 必须先做 betadisper
    □ betadisper 显著 → 需要单独讨论离散度差异

□ 5. β 多样性分解
    □ 考虑同时跑 Baselga 和 Podani
    □ 如果结论不一致 → 在 Discussion 中解释原因

□ 6. 功能多样性
    □ 性状是否标准化了？
    □ 物种数 > 性状数 + 1？（FRic 的前提）
    □ 有缺失性状吗？
    □ 报告 FDis 作为 FRic 的稳健替代

□ 7. 系统发育多样性
    □ 树是超度量树吗？
    □ 报告中使用了哪种零模型 + 为什么？
    □ NRI 和 NTI 的正负号是否与生态假设一致？
```
